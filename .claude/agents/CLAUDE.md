# Agent Governance

## Shared Rules for All Agents

### Output Discipline
- All output goes to `outputs/YYYYMMDD_<package_name>/` subdirectories
- Create the output directory if it doesn't exist
- Write structured JSON for findings, plain text for logs
- Never overwrite existing output - append or create new timestamped files

### Finding Format
Every finding must include:
```json
{
  "id": "AGENT-NNN",
  "title": "Short description",
  "severity": "critical|high|medium|low|info",
  "confidence": "certain|high|medium|low",
  "category": "component_exposure|data_storage|crypto|network|injection|...",
  "evidence": { "description": "...", "code_or_output": "..." },
  "exploitation": { "method": "adb|frida|poc_app", "steps": ["..."] },
  "remediation": "How to fix",
  "references": ["CWE-xxx", "OWASP Mobile Top 10"]
}
```

### Agent Interaction
- Orchestrator delegates; executors execute. No executor should delegate to other agents.
- When receiving findings from a prior phase, validate them before building on them.
- If a tool/command fails, try an alternative approach before reporting failure.
- Log all commands executed and their output for reproducibility.

### Device Interaction
- Device IP is always provided by the user or orchestrator - never auto-detect.
- Always check device connectivity before running device commands.
- If root is needed and device is not rooted, report which checks are skipped.
- Never install system-level tools on the device without explicit user approval.

### Security
- Never exfiltrate data from the device to external services.
- All captured data stays in the local outputs/ directory.
- Sanitize any credentials found - report their existence, not their values (unless needed for exploitation proof).
